Welcome to the gRoot visualization console!

gRoot Admin Console Visualizer

Interactively displays organizational units, groups, and users within a G Suite domain. Allows creation, modification,and deletion of G Suite Admin Console data.

Visualizations

Using the d3.js library, organizational units are visualized using a collapsible tree diagram, groups using circle-packing, and users using tree maps. The visualizations also incorporate searching and filtering for toggling granularity.

Admin Console

The application mainly uses the Google Admin SDK to retrieve organizational units, groups, and users and manipulate data and setting properties. It emulates Admin Console features and embeds them intuitively into the visualizations.

Open Source

The code is publicly available on Github, and instructions for cloning and launching the web application are included in the README.md file. The web application runs on the Google Cloud Platform (GCP) App Engine and Apache Maven.

Organizational Units

Conveys the OU hierarchy with an interactive tree visual (built with D3.JS) Domain, How many OUs / Layers Shown vs Total, Search, Limit Layers Create, Update, and Delete OUs given the necessary paths / info Auto Populating paths, Delete confirmation modal, Multiple Search match modal

Groups

A zoomable circle-packing graph with hovering tooltip informational cards provides an intuitive way to display all the groups in the domain and their direct and indirect parent-child relationships. The customer can also search and filter specific groups and toggle 4 different types of viewing options, as well as create, edit, and delete groups and modify their memberships.

Users

A zoomable treemap graph with rectangles representing org units and list of users inside each org unit A user detail page displaying the org unit path of the user and groups where the user is a member of. Users can be searched by name or email, filtered by org units / groups, ordered by first name / last name Customers can add user under specific org units directly, delete or rename users with ease. They could also change the org unit path or add /delete groups for the user.

Ready to get started?

If you have already set up your account and followed all the instructions below, sign in.

Clone the Repo and set up your own instance

1. Open up your terminal, we recommand Google Cloud Shell if you have a Google account, you can log in with your admin account.
2. $ cd to change directory to your target path, then clone the repo: $ git clone https://github.com/googleinterns/step45-2020.git
3. $ cd step45-2020
4. $ sudo apt install ./google-chrome-beta_current_amd64.deb
5. run $ mvn package appengine:run -DskipTests=true
6. After seeing Dev app Server is now running, click the rectangle button left to three dots in top right -> Preview on port 8080, the project is not set up yet, but you need to copy your server url, i.e. https://8080-5686c74b-aced-4f8e-83f7-f129248a983f.us-east1.cloudshell.dev
7. ctrl + c to interrupt the running project.

Cloud console configuration

Create the project

In the Cloud Console, first login to your test account, then select create your own project groot. Then, APIs & Services -> OAuth consent screen

Eable Admin SDK API

APIs & Services -> Dashboard -> ENABLE APIS AND SERVICES -> Search for Admin SDK -> ENABLE

OAuth Consent screen

1. APIs & Services -> OAuth consent screen
2. Select User Type to be Internal, then click Create.
3. Click Add scope, then enter the following scopes: https://www.googleapis.com/auth/admin.directory.orgunit, https://www.googleapis.com/auth/admin.directory.group, https://www.googleapis.com/auth/admin.directory.user
4. Under Authorized domains, add cloudshell.dev
5. Save the changes

Credential

1. APIs & Services -> Credentials -> Create Credentials -> OAuth Client ID -> Web client 1
2. Click the edit button
3. Under Authorized JavaScript origins -> URIs and Authorized redirect URIs -> URIs, add your server url, i.e. https://8080-5686c74b-aced-4f8e-83f7-f129248a983f.us-east1.cloudshell.dev

To run locally

run $ mvn package appengine:run -DskipTests=true or $ mvn package appengine:run (the later command will run test cases)
 * This runs your localhost:8080 cloudshell server
 * Copy the URL from the browser search bar, i.e. https://8080-5686c74b-aced-4f8e-83f7-f129248a983f.us-east1.cloudshell.dev
